拓展思考：模型的局限与进阶探索

通过前面的实验，我们成功地用线性回归模型预测了波士顿的房价。然而，在真实的工业界和学术研究中，跑通 Baseline 模型仅仅是一个开始。作为一个严谨的数据科学家，我们必须能够反思模型的局限性，并知道下一步该往哪里走。

1. 线性假设的局限与过拟合

线性回归做了一个非常强烈的假设：特征与房价之间是纯粹的直线关系。但在现实世界中，这种关系往往是非线性的。比如，房间数从 1 增加到 2 时房价可能大幅上涨，但从 10 增加到 11 时，房价可能就不会再涨那么多了（边际效益递减）。

为了捕捉非线性，我们可能会引入“多项式特征”（如 $RM^2, RM^3$）。但这又会引入机器学习中最致命的敌人：过拟合（Overfitting）。

2. 引入惩罚项：正则化 (Regularization)

当特征过多、模型过于复杂（发生过拟合）时，模型为了强行穿过每一个训练数据点，会导致权重参数 $w$ 变得极大。为了限制这种狂野的参数，统计学家在 MSE 损失函数的基础之上，加上了对参数大小的“惩罚项”，这就是正则化。

最常用的是 L2 正则化（岭回归，Ridge Regression），其新的损失函数如下：

$$ J(\mathbf{w}) = \text{MSE} + \lambda \sum_{j=1}^{d} w_j^2 $$

在这个公式中，$\lambda$ 是正则化强度。我们不仅要求预测误差 MSE 尽可能小，还要求权重 $w$ 尽可能接近于 0。这迫使模型变得平滑、简单，从而在未知的测试集上表现出更强的泛化能力。

3. 穿越时空的数据集：时代的局限

⚠️ 数据探索的盲区：著名的 50k 截断问题 (Censored Data)
波士顿房价数据集采集于 1978 年。如果你在做数据探索（EDA）时仔细观察过真实房价的直方图，你会发现：房价在刚好达到 $50,000 的地方，出现了一个异常突起的尖峰。
这不是因为刚好有那么多价值 5 万美元的房子，而是因为当年在统计时，凡是超过 5 万美元的房子，都被强制记录为了 5 万美元。这种数据在统计学上称为“截断数据（Censored Data）”。如果不把这些异常的 5 万美元样本剔除，你的线性回归曲线将会被严重地往下拉扯！

4. 工业界的下一步：拥抱生态

在本次实验中，我们为了理解底层数学原理，利用纯粹的 NumPy 手写了梯度下降和前向传播。但在真实的工业界业务中，为了追求极致的效率和稳定性，我们通常会直接调用高度封装的机器学习库，如 Scikit-Learn。

以下是用 Scikit-Learn 实现带正则化的岭回归（Ridge）的工业级写法：

# 引入 Scikit-Learn 中成熟的模型库
from sklearn.linear_model import Ridge
from sklearn.metrics import mean_squared_error, r2_score

# 1. 声明一个 L2 正则化的岭回归模型，alpha 即为惩罚系数 lambda
ridge_model = Ridge(alpha=1.0)

# 2. 拟合训练集（底层调用了高度优化的 C/Fortran 求解器，瞬间完成）

# 3. 在测试集上进行预测
