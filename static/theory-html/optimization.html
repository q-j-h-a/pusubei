核心算法：解析法与数值迭代法的博弈

在机器学习中，当我们定义好了模型 $f(x) = wx + b$ 和衡量误差的“尺子”——均方误差（MSE）后，接下来的核心挑战就是：如何找到那组最完美的斜率 $w$ 和截距 $b$，使得误差函数的取值达到全球最低？这在数学上被称为最优参数求解。针对波士顿房价预测这类线性任务，数学界提供了两条截然不同的思维路径。

1. 核心算法一：最小二乘法 (Ordinary Least Squares)

最小二乘法是一种“上帝视角”的解析解方法。它的名字来源于它的数学本质：寻找一组参数，使得预测值与真实值之差的平方（二乘）之和达到最小。

从微积分的角度来看，损失函数 $J(w, b)$ 是一个关于 $w$ 和 $b$ 的二元二次函数，其在坐标空间中呈现出一个开口向上的“碗状”表面。既然要找碗底，最直接的方法就是对 $w$ 和 $b$ 分别求偏导数，并令其导数等于 0。通过严密的代数推导，我们可以跳过所有中间尝试步骤，一步到位地算出标准答案。

对于本实验研究的一元线性回归，最优参数的解析公式如下（注意到 $w$ 与 $b$ 现在被紧密关联）：

$$ w = \frac{\sum_{i=1}^{m} (x^{(i)} - \bar{x})(y^{(i)} - \bar{y})}{\sum_{i=1}^{m} (x^{(i)} - \bar{x})^2} $$ $$ b = \bar{y} - w\bar{x} $$

利用 Python 的向量化运算，我们可以将上述求和符号转化为极其高效的代码实现：

import numpy as np

def ordinary_least_squares(x, y):
    """
    核心算法一：利用最小二乘法解析公式直接求解最优 w 和 b
    """
    x_mean = np.mean(x)
    y_mean = np.mean(y)
    
    # 分子：特征与标签的离差乘积之和
    numerator = np.sum((x - x_mean) * (y - y_mean))
    # 分母：特征离差的平方和
    denominator = np.sum((x - x_mean) ** 2)
    
    w = numerator / denominator
    b = y_mean - w * x_mean
    
    return w, b
延伸：解析解的代价
最小二乘法虽然精准且快速，但它在处理超大规模数据时会面临严重的算力瓶颈。当特征维度 $d$ 变得极高时，公式中涉及的矩阵求逆运算，其计算复杂度会呈立方级（$O(d^3)$）爆炸增长。这正是为什么我们需要引入更具普适性的数值解法。

2. 核心算法二：梯度下降 (Gradient Descent)

梯度下降是现代人工智能的动力引擎。不同于解析法的“一步到位”，它采用的是一种数值迭代的策略。我们可以把它想象成一个“蒙眼下坡”的过程：你被困在浓雾笼罩的山坡上，目标是走到谷底（误差最小点）。由于看不见全局，你只能用脚感受周围地面的坡度（计算梯度），然后朝着最陡峭的下坡方向迈出一小步，重复此过程直到抵达平坦的终点。

在数学上，梯度的方向指向函数值增长最快的方向。因此，为了减小误差，我们需要沿着负梯度的方向更新参数。更新步长由一个关键参数 $\alpha$（学习率）控制：

$$ w \leftarrow w - \alpha \frac{\partial J}{\partial w}, \quad b \leftarrow b - \alpha \frac{\partial J}{\partial b} $$

在代码实现中，我们需要在一个循环内不断重复这一过程。每一次循环（Epoch）模型都会变得更聪明一点：

def gradient_descent(x, y, w, b, learning_rate):
    """
    核心算法二：执行单步梯度下降更新
    """
    m = len(x)
    
    # 1. 前向预测：基于当前参数计算结果
    y_pred = w * x + b
    # 2. 计算预测残差
    error = y_pred - y
    
    # 3. 计算偏导数：即指示“下山方向”的梯度
    dw = (1 / m) * np.sum(error * x)
    db = (1 / m) * np.sum(error)
    
    # 4. 更新参数：朝着减小误差的方向迈出一小步
    w = w - learning_rate * dw
    b = b - learning_rate * db
    
    return w, b
工程避坑：学习率过大导致 NaN 报错 学习率 $\alpha$ 决定了下山的胆量。如果步子太小，收敛极其缓慢；如果步子太大，可能会“一步跨过碗底”并直接飞出误差曲面，导致误差值呈指数级增长，最终在 Python 中抛出数值溢出（NaN）错误。

3. 损失下降曲线 (Loss Curve)

在采用梯度下降时，实时观察损失曲线是判断模型是否训练正常的唯一标准。一个健康的训练过程，其均方误差（MSE）应该在初始阶段快速下降，随后逐渐平缓，标志着模型已经捕捉到了房价波动的本质规律。
