模型定义：一元线性映射与参数的业务逻辑

在完成了数据探索与预处理后，我们得到了纯净的自变量 $x$（特征：例如房屋的房间数）和因变量 $y$（标签：真实的房屋价格）。现在的核心任务是：构建一个数学上的“映射机器”，让计算机能够根据我们输入的任意 $x$，自动输出一个预测的房价 $\hat{y}$。

基于奥卡姆剃刀原理（如无必要，勿增实体），在探索未知规律时，我们总是从最简单的假设开始。因此，我们假设房价与房间数之间存在着最基础的线性关系（Linearity）。

1. 数学抽象：构造一元线性方程

在二维空间中，任何一条直线都可以被两个参数完全确定：斜率和截距。在机器学习中，我们将其写为如下标准形式：

$$ f(x) = wx + b $$
参数的深层业务含义与经济学直觉
在这条看似简单的初中代数公式背后，隐藏着极其直观的业务逻辑：
• 权重参数 $w$ (Weight)： 在经济学上相当于“边际溢价”。它代表着特征 $x$ 每增加一个单位（即多建一个房间），房屋的整体价格预计会上涨多少。它决定了整条预测直线的“倾斜程度”。
• 偏置参数 $b$ (Bias)： 相当于方程的“截距”。在业务上，它可以被理解为该地区房屋的“基础底价”或“地皮价值”。即使极端假设房间数 $x=0$，这套房产也有一个基础的地段价值 $b$。

2. 从单一样本到批量计算 (Vectorization)

在现实的工业实战中，我们绝不会使用极其低效的 Python for 循环去一条一条地计算 506 套房子的预测价格。由于 Python 的核心计算库 NumPy 具备强大的数组广播机制（Broadcasting），我们可以将整个包含 $m$ 个样本的特征向量 $\mathbf{x}$ 一次性代入公式。

此时，模型输出的将不再是一个单独的标量，而是一个包含了所有样本预测结果的预测向量 $\mathbf{\hat{y}}$：

$$ \mathbf{\hat{y}} = w\mathbf{x} + b $$
🎓 符号规范提醒：
在学术界，我们通常用带帽子的 $\hat{y}$（y-hat）来表示模型“估算/预测出来的值”，以此来严格区分数据集中原本“真实的标签值” $y$。接下来的所有优化算法，其终极目的都是在努力缩小 $\hat{y}$ 与 $y$ 之间的差距。

3. 代码实现：前向传播 (Forward Propagation)

将上述严谨的数学逻辑转化为代码其实非常简单。借助于 NumPy，我们只需要编写一行极简的代码，就能瞬间完成海量数据的前向预测计算：

import numpy as np

def predict(x, w, b):
    """
    一元线性回归的模型定义 (前向计算)
    
    参数:
    x: numpy 数组, 包含所有样本的单一特征 (如所有房子的房间数)
    w: 浮点数, 权重参数 (斜率)
    b: 浮点数, 偏置参数 (截距)
    
    返回:
    y_pred: numpy 数组, 包含所有样本的预测房价
    """
    # 利用 NumPy 的自动广播机制，一次性对所有样本进行标量乘法与加法计算
    y_pred = w * x + b
    
    return y_pred
