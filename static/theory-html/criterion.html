损失函数：量化模型的经验风险

在上一节中，我们建立了特征空间到房价的线性映射关系。但是，当我们随机初始化一组权重参数 $\mathbf{w}$ 时，模型最初的预测结果往往“错得离谱”。为了让机器“知道”自己预测得有多糟糕，我们需要定义一把尺子来衡量预测偏差，这把尺子就是损失函数（Loss Function）。

1. 均方误差 (Mean Squared Error, MSE)

在连续数值的回归问题中，最经典、最常用的经验风险指标是均方误差（MSE）。假设训练集包含 $m$ 个样本，损失函数 $J(\mathbf{w})$ 定义为所有样本预测值 $\hat{y}$ 与真实房价 $y$ 差值的平方和的平均值：

$$ J(\mathbf{w}) = \frac{1}{2m} \sum_{i=1}^{m} \left( \hat{y}^{(i)} - y^{(i)} \right)^2 $$
💡 工程细节：为什么公式里有一个 $\frac{1}{2}$ ？
严格意义上的均方误差是除以 $m$。这里额外乘以 $\frac{1}{2}$ 纯粹是为了数学上的便利：当我们在下一步对该函数求导时，平方项放下来的常数 2 刚好可以和底下的 2 抵消，使得梯度的代数表达更加简洁。这并不会改变最优解的位置。

结合我们上一节学到的矩阵乘法形式 $\mathbf{\hat{y}} = \mathbf{X}\mathbf{w}$，我们可以将损失函数写成极为紧凑的纯向量内积形式：

$$ J(\mathbf{w}) = \frac{1}{2m} (\mathbf{X}\mathbf{w} - \mathbf{y})^T (\mathbf{X}\mathbf{w} - \mathbf{y}) $$

2. 统计学深度：为什么非要用“平方”？

初学者常常会问：衡量误差，直接用绝对值差不好吗？为什么一定要多此一举去求平方？

从直观上看，平方操作不仅消除了正负号互相抵消的问题，还能成倍地放大“错得最离谱”的异常样本，从而迫使模型优先去修正那些大误差。

🧠 极大似然估计视角 (Maximum Likelihood Estimation)
从更深层的统计学假设来看：假设真实房价与我们模型的线性预测值之间存在的误差服从均值为 0 的高斯分布（正态分布）。根据中心极限定理，这是一个非常合理的假设。
当我们尝试去最大化观测数据出现的概率时（即进行 MLE 推导），最大化高斯分布的对数似然函数，在数学上完全等价于最小化均方误差。这就是 MSE 背后极其坚实的统计学基石。

3. 视觉直觉：美妙的凸函数

MSE 还有一个极其重要的代数性质：它是一个严格的凸函数（Convex Function）。这就意味着它的误差曲面就像一个底部平滑的碗。无论你从哪个初始点开始训练，只要顺着坡度往下走，你最终找到的局部极小值，必然也是全局唯一的最小值。

4. 代码实现：计算 Loss

借由 NumPy 强大的广播机制（Broadcasting）和矩阵运算，计算这 506 套房子的均方误差，仅仅需要一行极其凝练的核心逻辑：

import numpy as np

def compute_loss(X, y, w):
    """
    计算线性回归模型的均方误差 (MSE Loss)
    
    参数:
    X: numpy.ndarray, 形状 (m, d+1) 的特征矩阵
    y: numpy.ndarray, 形状 (m, 1) 的真实标签向量
    w: numpy.ndarray, 形状 (d+1, 1) 的权重向量
    """
    m = len(y)
    
    # 1. 矩阵乘法计算所有样本的预测值
    y_pred = np.dot(X, w)
    
    # 2. 计算误差平方和的均值
    # (y_pred - y) 产生一个 (m, 1) 的误差向量
    # ** 2 对向量内的每一个元素独立进行平方
    loss = (1 / (2 * m)) * np.sum((y_pred - y) ** 2)
    
    return loss
