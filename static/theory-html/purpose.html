实验目的：核心目标与能力矩阵

本次实验分为以下三个核心维度。

01 / Theory

数学原理与底层逻辑

理解线性回归的数学原理、损失函数与学习率的意义，并掌握最小二乘法与梯度下降两种求解路径。

02 / Workflow

工程落地与工作流

熟练掌握模型预测流程与数据预处理方法，理解特征标准化如何帮助模型稳定训练与更快收敛。

03 / Evaluation

模型评估与泛化测试

学会评估回归模型的性能指标，能够在未知数据上检验模型泛化能力并识别边界与局限。

1. 数学原理与底层逻辑

这一部分的重点不是记住结论，而是建立“模型为什么这样工作”的直觉。你需要把线性回归看成一个在特征空间中寻找最优直线的过程，理解预测值、真实值和误差之间的关系，并把损失函数视为衡量模型好坏的统一标准。

  • 深刻理解线性回归的数学原理及其在特征空间中的几何意义。
  • 掌握损失函数（如 MSE）与学习率（Learning Rate）在模型寻优过程中的具体含义。
  • 深入理解并掌握求解最优参数的两大核心方法：最小二乘法（解析解）与梯度下降优化方法（数值解）。
学习建议
这一部分建议结合图像和公式一起看。先理解“误差为什么会被平方”，再理解“为什么要沿着负梯度方向更新参数”，最后再回头看代码实现，会更容易把抽象概念和具体计算对应起来。

2. 工程落地与工作流

机器学习并不只是公式推导，更重要的是把一套标准流程真正跑通。本实验会引导你从数据定义开始，逐步完成前向预测、误差计算、参数更新和结果展示，形成完整的工程闭环。

  • 熟练掌握机器学习模型预测的基本流程，涵盖数据定义、前向传播、误差计算与反向更新。
  • 掌握数据预处理（如特征标准化 Z-score）的具体意义，理解其如何避免模型被大数值特征带偏并加速算法收敛。
  • 能够从页面交互、图表变化和数值输出中，读懂每一步训练过程所发生的变化。
工程视角下的关键点 一次完整的实验不是“跑出一个结果”这么简单，而是要理解每个阶段的输入、输出与作用。只要你能把数据流、参数流和可视化结果串起来，就已经掌握了这类实验最核心的工程方法。

3. 模型评估与泛化测试

一个模型是否真正有用，不能只看它在训练集上的表现，还要看它在未知数据上的稳定性。因此，评估指标与泛化能力是本实验的最后一块拼图，也是检验模型是否“学会了规律”而不是“记住了答案”的关键。

  • 学会评估回归模型的性能指标，如均方根误差 RMSE、决定系数 $R^2$。
  • 能够脱离训练集，在未知的测试集上客观度量模型的泛化能力。
  • 识别模型的边界与局限，理解为什么评估结果与数据质量、特征选择和训练策略密切相关。
最终目标
当你完成这一实验后，应当不仅能“看懂结果”，还能够“解释结果为什么会这样”，并知道下一步该从数据、特征、参数还是优化策略上继续改进。
